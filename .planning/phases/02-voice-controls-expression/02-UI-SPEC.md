## Layout Contract

### Overall Structure

The app renders as a single column, max-width 600px, centered. Five labeled sections stack vertically with `xl` (32px) gaps between them. Each section has a `lg` (24px) internal padding, a 1px `#4a4a6a` border, and a `6px` border-radius.

### Section Order (top to bottom)

1. **Transport** — play/stop button, volume slider, mute toggle (horizontal row)
2. **Pitch** — piano keyboard visual with overlaid vertical slider, pitch readout below
3. **Voice** — horizontal chip group for voice-type presets
4. **Phonation** — horizontal button group for phonation modes
5. **Expression** — vibrato rate slider, vibrato extent slider, jitter amount slider (vertical stack)

### Section 1: Transport

- Horizontal flex row, `align-items: center`, gap `md` (16px)
- Play/stop button: 44x44px, square, centered icon, accent background when playing
- Volume slider: flex-grow, native `<input type="range">`, min 0 max 1 step 0.01
- Volume label: "Volume" above slider in `Label` typography, current percentage right-aligned
- Mute toggle: 44x44px square button, shows unmute icon normally, shows red mute icon when muted

### Section 2: Pitch

- Section heading: "Pitch"
- Piano keyboard visual: SVG, 2 octaves visible (matching QWERTY range from D-03), white keys `#d4d4d8`, black keys `#27272a`, key width minimum 44px for touch targets
- Vertical pitch slider overlaid on the piano: translucent track, thumb snaps to nearest semitone visually but value is continuous (log-scale per D-01)
- QWERTY key labels: rendered inside each piano key in `Label` typography, toggled by a small icon button in the section header (keyboard icon)
- Pitch readout below piano: single line, `Readout` typography, format per D-05: `220 Hz . A3 . +0c` with the note name in accent color
- f0 range: 55-1100 Hz per D-02

### Section 3: Voice

- Section heading: "Voice"
- Horizontal flex row with wrap, gap `sm` (8px)
- Seven chip buttons: Soprano, Mezzo, Alto, Tenor, Baritone, Bass, Child
- Chip dimensions: padding `8px 16px`, border-radius `16px` (pill shape)
- Default state: secondary background, border color, text primary
- Selected state: active/selected bg, accent border (2px), accent text
- Only one chip active at a time (radio behavior)

### Section 4: Phonation

- Section heading: "Phonation"
- Horizontal flex row, gap `sm` (8px)
- Four buttons: Breathy, Modal, Flow, Pressed (per D-09)
- Same styling as voice chips: pill shape, radio behavior, accent on active
- Modal is the default selected state

### Section 5: Expression

- Section heading: "Expression"
- Three labeled sliders stacked vertically, gap `md` (16px)
- Each slider row: label left, value readout right, `<input type="range">` below spanning full width
- Vibrato Rate: label "Vibrato Rate", range 0-10 Hz, step 0.1, default 5 Hz (D-07), readout "{value} Hz"
- Vibrato Extent: label "Vibrato Extent", range 0-100 cents, step 1, default 10 cents (D-07), readout "{value} cents"
- Jitter Amount: label "Jitter", range 0-1, step 0.01, default 0, readout "{value}" (two decimal places)

## Interaction Contract

### QWERTY Keyboard Piano (D-03)

- `keydown` events on `document` when audio is playing
- Bottom two rows: Z=C, S=C#, X=D, D=D#, C=E, F=F, V=F#, G=G, B=G#, N=A, J=A#, M=B (lower octave); Q=C, 2=C#, W=D, 3=D#, E=E, 4=F, R=F#, 5=G, T=G#, 6=A, Y=A#, 7=B (upper octave)
- On keydown: set `voiceParams.f0` to the corresponding note frequency, highlight the key visually with accent color
- On keyup: remove visual highlight (f0 stays at last pressed note — no note-off behavior)
- Ignore repeat events (`event.repeat === true`)

### Pitch Slider

- `input` event on range input updates `voiceParams.f0` continuously
- Log-scale mapping: slider position 0-1 maps to 55-1100 Hz logarithmically so each semitone occupies equal slider distance
- No debounce — direct write to store, bridge handles smoothing via `setTargetAtTime`
- `touch-action: none` on the slider element

### Voice Preset Loading (D-13)

- Clicking a voice chip: sets f0 to voice default, sets formant frequencies to voice defaults, resets phonation to Modal
- This is a "complete voice reset" — all expression params (vibrato, jitter) are NOT reset (only f0, formants, phonation)
- Transition: store writes are synchronous, audio smooths via `setTargetAtTime`

### Phonation Mode Switch (D-11)

- Clicking a phonation button: sets openQuotient, aspirationLevel, and spectralTilt to the mode's preset values
- No transition animation — instant parameter update, audio engine smooths

### Transport Controls (D-14, D-15)

- Play button: if AudioContext not initialized, initialize on click (user gesture). Toggle play/stop.
- Mute button: toggles `voiceParams.muted`. When muted, master gain goes to 0 via `setTargetAtTime`. Volume slider position is preserved.
- All transport actions respond within one animation frame.

## States

### Initial State (app load)

- Transport: stopped, volume at 50%, unmuted
- Pitch: f0 at 120 Hz (current default), readout shows "120 Hz . B2 . +0c" (approximately)
- Voice: no preset selected (custom state from Phase 1 defaults)
- Phonation: Modal selected
- Expression: vibrato rate 5 Hz, vibrato extent 10 cents, jitter 0

### Playing State

- Play button shows stop icon, accent background
- All sliders and controls remain interactive
- Pitch readout updates in real time as f0 changes

### Muted State

- Mute button shows muted icon in red (`#ef4444`)
- Volume slider is visually dimmed (opacity 0.5) but still draggable
- Audio output is silent

## Phase-Specific Notes

### What This Phase Does NOT Include

- No F1/F2 vowel chart (Phase 3)
- No piano harmonics visualization (Phase 3)
- No formant range overlays (Phase 3)
- No vocal strategies (Phase 4)
- No responsive layout optimization (Phase 6)
- No tooltips on controls (Phase 6)
- No expert mode / advanced parameter sliders (Phase 6)

### CSS Strategy

Continue the Phase 1 pattern: all styles in `src/app.css` using plain CSS with CSS custom properties for the color tokens. No CSS modules, no Tailwind, no CSS-in-JS. Extract component-scoped styles into `<style>` blocks in `.svelte` files as components are created.

Recommended CSS custom properties to add to `:root`:

```css
--color-bg: #1a1a2e;
--color-surface: #2a2a4a;
--color-accent: #6366f1;
--color-text: #e0e0e0;
--color-text-secondary: #8a8aaa;
--color-border: #4a4a6a;
--color-hover: #3a3a5a;
--color-active: #3a3a6a;
--color-muted: #ef4444;

--spacing-md: 16px;
```
