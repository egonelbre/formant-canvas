**Linked update verification (LINK-01):**
The chain is:
1. User drags on VowelChart -> writes voiceParams.f1Freq, voiceParams.f2Freq (synchronous)
2. Svelte 5 batches the writes in the same microtask
3. $derived in PianoHarmonics recomputes harmonic amplitudes (same frame)
4. $derived in VowelChart updates handle position and active region (same frame)
5. $effect in App.svelte fires bridge.syncParams() -> audio params update via setTargetAtTime

No requestAnimationFrame, no setTimeout, no async -- all synchronous reactive propagation.

**LINK-03 (no glitches while dragging):**
Already handled by AudioBridge.syncParams() using setTargetAtTime with ~20ms time constant on all BiquadFilterNode frequency/Q parameters. No new code needed -- just verify the existing mechanism works with the new drag source.
