## Layout Contract

### Overall Structure

The app layout expands from the Phase 2 single-column (600px max) to a wider layout accommodating the visualizations. The new layout is a single column, max-width 960px, centered. The existing control sections from Phase 2 remain below the new visualization sections.

### Section Order (top to bottom)

1. **Transport** (existing, unchanged)
2. **Piano Harmonics** (NEW) -- 5-octave piano with harmonic bars and formant curves
3. **F1/F2 Vowel Chart** (NEW) -- Hillenbrand background, drag handle, voice-type overlays
4. **Pitch** (existing, adapted -- remove piano keyboard from this section since it moves to Piano Harmonics)
5. **Voice** (existing, unchanged)
6. **Phonation** (existing, unchanged)
7. **Expression** (existing, unchanged)

### Section 2: Piano Harmonics (NEW)

**Dimensions:** Full width of the 960px column. SVG viewBox height is 200px (120px for keys + 80px for harmonic bars/formant curves above keys).

**Piano keyboard (bottom 120px of SVG):**
- 5 octaves C2-B6: 35 white keys at 14px each = 490px logical width, scaled to fill container via `preserveAspectRatio="xMidYMid meet"`
- White keys: `#d4d4d8` fill, `#888888` 0.5px stroke
- Black keys: `#27272a` fill, `#333333` 0.5px stroke, 9px wide, 52px tall (proportional to Phase 2 ratios)
- Current f0 key: accent fill (`#6366f1`)
- C-labels: "C2", "C3", "C4", "C5", "C6" rendered below each C key in axis tick label typography (11px)

**Harmonic bars (above keys, 0-80px region):**
- Each harmonic drawn as a vertical bar centered on its corresponding piano key position
- Bar width: 4px (thin enough to avoid overlap at close harmonics)
- Bar height: proportional to amplitude after analytic formant filter evaluation, max height 72px (leaving 8px margin from top)
- Fundamental bar: accent color (`#6366f1`), 100% opacity
- Overtone bars: `#e0e0e0` at 80% opacity
- At least 12 harmonics rendered (D-06). Harmonics beyond the piano range (>B6) are not drawn.
- Amplitude calculation: `A(f) = sum_i[ gain_i / sqrt(1 + (f - fc_i)^2 / bw_i^2) ]` for each formant i, evaluated at harmonic frequency f. Use `bandwidthToQ` from existing `formant-utils.ts`.

**Formant response curves (overlaid on harmonic region):**
- Four continuous curves (F1-F4) drawn as SVG `<path>` elements across the frequency range
- Each curve: 2px stroke, formant color (F1 orange, F2 green, F3 blue, F4 purple), no fill
- Opacity: 70% to avoid obscuring harmonic bars
- Curves are computed by evaluating each formant's bandpass response at ~200 evenly spaced frequencies across C2-B6 range
- Y-axis: same as harmonic bars (amplitude 0 at key top, max at top of harmonic region)

**Formant center markers:**
- Four vertical dashed lines (2px dash, 2px gap) at F1-F4 center frequencies, extending from top of harmonic region down to the key
- Each line in its formant color at 50% opacity
- Small label ("F1", "F2", "F3", "F4") at the top of each line in formant legend label typography (11px, semibold), formant color

**Interaction:** Click or drag on a piano key sets `voiceParams.f0` to that note's frequency. Uses the same pointer-capture drag pattern as Phase 2 PianoKeyboard. `touch-action: none` on the SVG element.

### Section 3: F1/F2 Vowel Chart (NEW)

**Dimensions:** 480px x 400px SVG, centered within the 960px column. Chart plot area has 48px left margin (Y-axis labels), 24px right margin, 32px top margin, 40px bottom margin (X-axis labels + citation).

**Axes (D-01, D-02):**
- X-axis: F2 in Hz, increasing rightward. Log scale via `d3-scale` `scaleLog()`. Range: 600 Hz to 3000 Hz.
- Y-axis: F1 in Hz, increasing upward (Cartesian, NOT phonetics-inverted per D-01). Log scale. Range: 200 Hz to 1000 Hz.
- Axis lines: `#4a4a6a`, 1px solid
- Tick marks: major ticks at round Hz values (F1: 200, 300, 400, 500, 600, 700, 800, 1000; F2: 600, 800, 1000, 1500, 2000, 2500, 3000). Tick length 6px outward.
- Tick labels: axis tick typography (11px), `#8a8aaa` (text-secondary)
- Grid lines: `#4a4a6a` at 40% opacity, 0.5px, at each major tick
- Axis titles: "F1 (Hz)" on Y-axis (rotated 90 degrees CCW), "F2 (Hz)" on X-axis, in label typography (12px, semibold)

**Hillenbrand ellipses (D-16):**
- 12 vowel regions drawn as `<ellipse>` SVG elements
- Each ellipse: center at Hillenbrand mean F1/F2, radii from 1 standard deviation in F1 and F2 dimensions
- Fill: `#e0e0e0` at 6% opacity
- Stroke: `#e0e0e0` at 20% opacity, 1px
- IPA label: rendered at each ellipse center in IPA symbol typography (14px, semibold), `#8a8aaa` (text-secondary)
- Labels use IPA Unicode characters: /i/, /e/, /ae/, /a/, /o/, /u/, etc. (full 12-vowel Hillenbrand set)

**Active vowel region highlight (D-19, RANGE-03):**
- The ellipse region that contains the current F1/F2 position gets a brighter fill: accent at 15% opacity
- Hit test: point-in-ellipse check using `((x-cx)/rx)^2 + ((y-cy)/ry)^2 <= 1`
- If the handle is inside multiple overlapping ellipses, highlight the smallest (most specific)

**Per-voice-type formant range overlays (D-17, RANGE-02):**
- Three overlay regions (male, female, child) showing the convex hull of all Hillenbrand vowel means for that speaker group
- Drawn as `<polygon>` or `<path>` with the voice-type overlay colors (blue/orange/green at 10% opacity)
- Stroke: same color at 30% opacity, 1px dashed (4px dash, 4px gap)
- Only one overlay visible at a time. Default: none shown.
- Voice-type overlay selector: a 3-chip group ("Male Range", "Female Range", "Child Range") positioned above the chart, right-aligned. Uses existing ChipGroup component pattern. An additional "None" option (default) hides all overlays.

**Drag handle (D-12, D-13, VOWEL-03):**
- Filled circle, 16px diameter, accent color (`#6366f1`), no stroke
- Positioned at current `voiceParams.f1Freq` (Y) and `voiceParams.f2Freq` (X) on the chart
- Cursor: `grab` at rest, `grabbing` while dragging

**Vowel preset positions (D-09, D-10, D-11):**
- IPA symbols rendered at their Hillenbrand centroid positions on the chart
- Clickable: clicking an IPA symbol snaps `voiceParams.f1Freq` and `voiceParams.f2Freq` (and f3Freq, f4Freq) to that vowel's formant values
- Click target: invisible `<circle>` with 20px radius centered on each symbol for comfortable touch/click
- Hover state: IPA symbol color transitions from `#8a8aaa` to `#e0e0e0`
- The drag handle overlaps with a vowel position when snapped; the handle renders on top

**Citation (D-18, VOWEL-05):**
- Text "Data: Hillenbrand et al. (1995)" rendered below the X-axis in citation typography (11px, regular)
- Color: `#8a8aaa` (text-secondary)
- Position: right-aligned within the plot area, 4px below the X-axis title

## Interaction Contract

### F1/F2 Chart Drag (D-12, D-13, D-14, D-15)

- `pointerdown` on drag handle: set pointer capture, switch cursor to `grabbing`
- `pointermove` while captured: convert pointer coordinates to F1/F2 Hz via inverse log scale, clamp to axis ranges (F1: 200-1000 Hz, F2: 600-3000 Hz), write directly to `voiceParams.f1Freq` and `voiceParams.f2Freq`
- F3/F4 tracking: when F1/F2 are dragged, F3 stays at its current value, F4 stays at its current value. (F3/F4 are independent in this phase; vocal strategy tracking is Phase 4.)
- `pointerup` / `pointercancel`: release capture, restore `grab` cursor
- `touch-action: none` on the chart SVG element
- No debounce -- direct store writes at pointer event rate. AudioBridge smooths via `setTargetAtTime` (LINK-03).
- Performance target: all views (audio, F1/F2 chart position, piano harmonic amplitudes) update within one `requestAnimationFrame` cycle (LINK-01).

### Vowel Preset Click (D-09, D-10, D-11)

- Clicking an IPA symbol on the F1/F2 chart sets:
  - `voiceParams.f1Freq` to Hillenbrand mean F1
  - `voiceParams.f2Freq` to Hillenbrand mean F2
  - `voiceParams.f3Freq` to Hillenbrand mean F3
  - `voiceParams.f4Freq` to Hillenbrand mean F4 (if available; otherwise keep current)
- Bandwidth values (f1BW, f2BW, etc.) are NOT changed by vowel preset click
- The drag handle animates to the new position (no transition -- instant snap)

### Piano Click-to-Tune (PIANO-05)

- Click or drag on piano key area sets `voiceParams.f0` to `midiToHz(midi)` for the clicked key
- Uses identical pointer-capture pattern from Phase 2 PianoKeyboard
- This replaces the Phase 2 pitch section's embedded piano for f0 control

### Voice-Type Overlay Toggle

- Clicking a voice-type chip ("None", "Male Range", "Female Range", "Child Range") shows/hides the corresponding formant range overlay on the F1/F2 chart
- Radio behavior: only one overlay visible at a time
- "None" is the default selected state

### Linked Update Verification (LINK-01, LINK-03)

All parameter changes from ANY source (F1/F2 drag, vowel preset click, piano click, pitch slider, voice preset, phonation switch) must:
1. Write to `voiceParams` (single source of truth, LINK-02)
2. Trigger `$effect` in App.svelte which calls `bridge.syncParams()`
3. Trigger `$derived` recomputation in all visualization components
4. Complete audio parameter update + visual re-render within one animation frame

- Handle position jumps to preset centroid
- Harmonic bars update to reflect new formant positions
- Active vowel region highlights the snapped vowel's ellipse

### Voice-Type Overlay Visible State

- Selected chip shows accent border/text
- Corresponding overlay polygon visible on chart with dashed border
- Other overlays hidden

## Phase-Specific Notes

### What This Phase Does NOT Include

- No vocal strategy overlays on piano or F1/F2 chart (Phase 4)
- No auto-tune / locked formant tracking (Phase 4)
- No URL sharing or undo/redo (Phase 5)
- No tooltips, progressive disclosure, or expert mode (Phase 6)
- No responsive layout below 960px (Phase 6)
- No spectrogram, waveform, or spectrum views (v2)
- No F1/F2 trail/path display (v2)

### CSS Strategy

Continue existing pattern: global tokens in `src/app.css`, component-scoped styles in `<style>` blocks. Add `--color-f1` through `--color-f4` tokens to `:root`. Increase `main` max-width from 600px to 960px.

### Performance Contract (LINK-01, LINK-03)

- Harmonic amplitude computation is pure math (`$derived` from voiceParams) -- no async, no Web Audio API calls
- Formant response curves recompute on formant parameter change only (not on every f0 change)
- SVG element count budget: PianoHarmonics ~120 elements (35 white keys + 25 black keys + 12 harmonic bars + 4 curves + 4 markers + labels), VowelChart ~60 elements (12 ellipses + 12 labels + handle + axes + ticks + overlays). Total well under 200 per component.
- No `requestAnimationFrame` loop needed -- Svelte reactivity handles updates synchronously within the microtask that writes to `$state`

### Accessibility Notes

- Piano SVG: `role="group"`, `aria-label="Piano keyboard with harmonics"`
- F1/F2 chart SVG: `role="img"`, `aria-label="F1/F2 vowel space diagram"`
- Drag handle: `role="slider"`, `aria-label="Current vowel position"`, `aria-valuemin`/`aria-valuemax` for F1/F2 ranges
- IPA vowel presets: `role="button"`, `aria-label="Set vowel to {IPA symbol}"`
- Voice-type chips: `role="radiogroup"` with `role="radio"` children
