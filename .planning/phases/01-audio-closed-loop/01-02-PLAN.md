Create `src/lib/audio/worklet/glottal-processor.ts` implementing an AudioWorkletProcessor named `'glottal-processor'`.

**CRITICAL: The worklet runs in a separate scope. It CANNOT import from other project files.** Inline the Rosenberg pulse math directly in this file (copy the formula from `rosenberg.ts`). Do NOT use `import` statements — `AudioWorkletGlobalScope` does not support ES module imports.

The processor must:

1. Maintain internal state: `phase` (0-1 normalized), `f0` (default 120), `aspirationLevel` (default 0.03), `openQuotient` (default 0.6)

2. Listen for parameter updates via `this.port.onmessage`:
   - `{ type: 'params', f0: number, aspirationLevel: number, openQuotient: number }`
   - Update internal state from the message data (only update fields that are present in the message)

3. In `process()` (128 samples per call):
   - For each sample: compute `rosenbergSample(this.phase, this.openQuotient)` using the inlined formula
   - Mix aspiration noise: `sample + (Math.random() * 2 - 1) * this.aspirationLevel` (per D-02)
   - Advance phase: `this.phase += this.f0 / sampleRate; if (this.phase >= 1.0) this.phase -= 1.0;`
   - Write to `outputs[0][0][i]`
   - ALWAYS return `true` (continuous source — per Pitfall 5)

4. End file with `registerProcessor('glottal-processor', GlottalProcessor);`

The Rosenberg C formula to inline (matching rosenberg.ts):
- `Tn = openQuotient`, `Tp = 0.4 * Tn`
- phase < Tp: `0.5 * (1 - Math.cos(Math.PI * phase / Tp))`
- phase < Tn: `Math.cos(Math.PI * 0.5 * ((phase - Tp) / (Tn - Tp)))`
- else: 0

**AudioBridge responsibilities:**

1. **`async init()`** — Creates `AudioContext`, loads the worklet processor via:
   ```typescript
   const workletUrl = new URL('./worklet/glottal-processor.ts', import.meta.url);
   await this.ctx.audioWorklet.addModule(workletUrl);
   ```
   **Vite 8 TS fallback:** If `addModule()` throws a `SyntaxError` at runtime (TypeScript syntax not transpiled — see RESEARCH.md Q1), catch the error and retry with the `.js` fallback path:
   ```typescript
   try {
     const workletUrl = new URL('./worklet/glottal-processor.ts', import.meta.url);
     await this.ctx.audioWorklet.addModule(workletUrl);
   } catch (e) {
     // Fallback: Vite did not transpile .ts for addModule — try .js
     const fallbackUrl = new URL('./worklet/glottal-processor.js', import.meta.url);
     await this.ctx.audioWorklet.addModule(fallbackUrl);
   }
   ```
   If the Task 1 executor already applied the .js fallback (renamed file), this try/catch is harmless — the .ts URL will 404 and the .js will succeed. If .ts works (expected), the catch never fires.

   Then builds the audio graph.

2. **Audio graph (per D-08 parallel topology):**
   ```
   GlottalWorkletNode --+--> BiquadF1(bandpass) --> GainF1 --+
                        +--> BiquadF2(bandpass) --> GainF2 --+--> SumGain --> MasterGain --> ctx.destination
                        +--> BiquadF3(bandpass) --> GainF3 --+
                        +--> BiquadF4(bandpass) --> GainF4 --+
   ```
   - 4 `BiquadFilterNode`s with `type: 'bandpass'`
   - 4 `GainNode`s for per-formant amplitude
   - 1 `GainNode` as sum node (just connects the 4 formant gains)
   - 1 `GainNode` as master volume
   - Each BiquadFilterNode's `.frequency` and `.Q` are set initially from VoiceParams defaults using `bandwidthToQ()` import

3. **`async resume()`** — Calls `this.ctx.resume()` (per D-06/AUDIO-08). Returns the AudioContext state.

4. **`syncParams()`** — Reads from `voiceParams` and forwards to Web Audio nodes. Uses `setTargetAtTime` for ALL parameter changes (per AUDIO-06). Time constants:
   - Formant frequency: tau = 0.02 (20ms)
   - Formant Q: tau = 0.02
   - Formant gain: tau = 0.01
   - Master gain: tau = 0.01
   
   For each formant i (0-3), forward:
   - `this.formants[i].frequency.setTargetAtTime(freqValue, now, 0.02)`
   - `this.formants[i].Q.setTargetAtTime(bandwidthToQ(freqValue, bwValue), now, 0.02)`
   - `this.formantGains[i].gain.setTargetAtTime(gainValue, now, 0.01)`
   
   Forward master gain:
   - `this.masterGain.gain.setTargetAtTime(voiceParams.masterGain, now, 0.01)`
   
   Forward worklet params via postMessage:
   - `this.workletNode.port.postMessage({ type: 'params', f0: voiceParams.f0, aspirationLevel: voiceParams.aspirationLevel, openQuotient: voiceParams.openQuotient })`

5. **`start()` / `stop()`** — Controls whether sound is playing. On `start()`: ensure context is resumed, connect workletNode to formant chain. On `stop()`: disconnect workletNode from formant chain (or set master gain to 0 with fast ramp, then disconnect).

6. **`destroy()`** — Closes the AudioContext.

CRITICAL: NEVER assign `audioParam.value = x` directly. ALWAYS use `setTargetAtTime`. This prevents zipper noise (AUDIO-06, Pitfall 3).
