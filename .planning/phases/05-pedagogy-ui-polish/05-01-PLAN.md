Create `src/lib/charts/strategy-chart-math.ts` with pure functions for R1/R2 Sundberg-style chart computation:

- `createPitchScale(minHz: number, maxHz: number, widthPx: number)` -- returns `d3.scaleLinear().domain([minHz, maxHz]).range([0, widthPx])`
- `createFreqScale(minHz: number, maxHz: number, heightPx: number)` -- returns `d3.scaleLinear().domain([minHz, maxHz]).range([heightPx, 0])` (inverted Y for SVG)
- `computeDiagonalLine(harmonic: number, xDomain: {min, max}, yDomain: {min, max})` -- returns `{x1, y1, x2, y2}` in Hz coordinates where y = harmonic * x, clamped to yDomain. For each endpoint: start with x, compute y = harmonic * x, clamp y to [yDomain.min, yDomain.max], then back-compute x = y/harmonic if clamped
- `pitchToNoteName(hz: number)` -- converts Hz to nearest note name string like 'C4', 'A4' using 12-TET, referencing A4=440
- `generateAxisTicks(minHz: number, maxHz: number, noteNames: string[])` -- returns array of `{hz, label}` for X-axis note labels (e.g. C3, C4, C5)

Import `scaleLinear` from `d3-scale`. Do NOT use scaleLog -- Sundberg reference charts use linear axes (per RESEARCH.md A1).
