## Layout Contract

This section extends the template for Phase 5's layout-heavy scope.

### CSS Grid Structure (D-01 through D-08)

```
+--------------------------------------------------------+
| header: [Voice chips]    [Expert toggle] [Transport]   |
+----------+--------------------------+------------------+
| sidebar: | harmonics:               | charts:          |
| Pitch    |   PianoHarmonics         |   C: VowelChart  |
| Vibrato  |   (formant curves +      |   B: R2 Strategy |
| Phonation|    harmonic bars)        |   A: R1 Strategy |
| Strategy |                          |                  |
+----------+--------------------------+------------------+
| piano: PianoKeyboard (full width, touch multi-touch)   |
+--------------------------------------------------------+
```

Grid definition:
- Rows: `auto 1fr auto` (header / main content / piano)
- Columns: `minmax(200px, 240px) 1fr minmax(240px, 300px)`
- Areas: `header header header` / `sidebar harmonics charts` / `piano piano piano`
- Gap: `0` (borders provide separation)
- Height: `100dvh` with `100vh` fallback
- Overflow: `hidden` on root

### Panel sizing at 1024x700 (UI-05)
- Sidebar: 200px (minimum)
- Charts column: 240px (minimum)
- Harmonics: 1024 - 200 - 240 = 584px (remaining)
- Header: auto height (~48px with chips + transport)
- Piano: auto height (~120px for keyboard)
- Main content area: 700 - 48 - 120 = ~532px for sidebar/harmonics/charts

### Right column panels (D-03)
- Panels A (R1), B (R2), C (VowelChart) stack vertically via `flex-direction: column`
- Each gets `flex: 1` for equal height distribution
- 1px `--color-border` divider between panels
- Independent Y-axis scales per panel (RESEARCH.md open question 1 resolution)

### Expert mode layout behavior (RESEARCH.md Pitfall 3)
- Expert controls appear inline within sidebar panels
- Sidebar has `overflow-y: auto` to handle overflow from extra controls
- Grid row heights do NOT change when expert mode toggles
- Piano keyboard position is stable regardless of expert mode state

## Interaction Contract

### Touch targets (D-21, UI-06)
- Minimum 44x44px for all interactive elements
- Piano keys: minimum 44px wide per white key at the narrowest viewport
- `?` tooltip trigger button: 44x44px tap area (visual icon can be smaller)
- Chip group items: minimum 44px height
- Slider thumb: 44px diameter touch area

### Pointer Events (D-22)
- All draggable elements: `touch-action: none` CSS property
- Pointer capture on `pointerdown`, release on `pointerup`/`pointercancel`
- Applies to: VowelChart drag handle, PianoKeyboard, slider thumbs, R1/R2 chart interactions (if any)

### Multi-touch piano (D-23)
- Track active pointers in `Map<pointerId, midiNote>`
- Visual feedback (highlight) on ALL touched keys simultaneously
- Last-touched pointer determines active f0 (monophonic)
- Release: remove pointer from map; if remaining pointers exist, last remaining sets f0

### Tooltip interaction (D-19)
- `?` button: `onclick` toggles popover open/closed (primary trigger for touch)
- Desktop convenience: `mouseenter` opens, `mouseleave` closes (does not interfere with click toggle)
- Click anywhere outside tooltip dismisses it
- Only one tooltip open at a time (opening one closes any other)
- Tooltip positions above trigger by default; flips below if insufficient space above

### Expert mode toggle (D-14)
- Toggle switch in header row, right-aligned near transport controls
- Default: OFF
- Toggles `expertMode` boolean (app-level UI state, NOT in VoiceParams)
- Transition: advanced controls appear/disappear without layout shift in main grid
