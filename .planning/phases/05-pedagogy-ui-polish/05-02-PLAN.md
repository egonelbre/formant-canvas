Create `src/lib/charts/R1StrategyChart.svelte` -- a Sundberg-style SVG chart showing R1 (first resonance) frequency vs pitch.

Props:
```typescript
interface Props {
  f0: number;                    // Current fundamental frequency in Hz
  f1Freq: number;                // Current F1 frequency for marker
  r1Strategy: string | null;     // Active R1 strategy ID
  strategyMode: string;          // 'off' | 'overlay' | 'locked'
  voicePreset: string | null;    // For formant range shading
  width?: number;                // Container width (default 280)
  height?: number;               // Container height (default 180)
}
```

**NOTE:** This component receives values as props from App.svelte, NOT by importing voiceParams directly. This keeps it a pure visualization component. App.svelte will pass `f0={voiceParams.f0}` etc.

SVG structure with margins (left: 45px for Y-axis labels, right: 10px, top: 20px for title, bottom: 25px for X-axis):
- Plot area = (width - 55) x (height - 45)

Axes (D-09):
- X-axis: Pitch in Hz, rendered as musical note names. Domain: C2 (65 Hz) to C6 (1047 Hz). Linear scale.
- Y-axis: Resonance frequency in Hz. Domain: 200 Hz to 1200 Hz (covers F1 range). Linear scale.
- Use `createPitchScale` and `createFreqScale` from `strategy-chart-math.ts` for scales.
- Draw 4-5 X-axis ticks at note positions (C2, C3, C4, C5, C6) using `generateAxisTicks`. Render as `<text>` elements.
- Draw 4-5 Y-axis ticks (200, 400, 600, 800, 1000, 1200 Hz). Render as `<text>` elements.
- Axis gridlines: `stroke: var(--color-border)` at `opacity: 0.5`, stroke-width 0.5.

Diagonal harmonic lines (D-09):
- For harmonics n=1,2,3: use `computeDiagonalLine(n, {min:65, max:1047}, {min:200, max:1200})` to get Hz endpoints, then convert to pixel coords via scales.
- Draw as `<line>` elements: `stroke: #e0e0e0`, `opacity: 0.4`, stroke-width: 1.
- Label each line at its right endpoint with the notation (f0, 2f0, 3f0) using `<text>` font-size 10px.

Shaded formant range region (D-10):
- If `voicePreset` is set, look up `VOICE_PRESETS[voicePreset]` to get F1 center and F1 bandwidth.
- Compute range as `[f1 - 2*f1BW, f1 + 2*f1BW]` (2 bandwidths above and below center covers typical range).
- Also consider the full range across all voice types: min F1 ~500 Hz to max F1 ~1100 Hz as a gentle background band.
- Draw as `<rect>` spanning full X, Y from range min to range max: `fill: var(--color-f1)`, `opacity: 0.15`.

Voice range bracket on X-axis (D-11):
- If `voicePreset` is set, draw a horizontal bracket showing that voice type's pitch range (from `VOICE_PRESETS[voicePreset].f0Default` +/- typical range). Use approximate ranges: soprano 262-1047, mezzo 196-784, alto 175-659, tenor 131-524, baritone 98-392, bass 65-262, child 262-784.
- Draw as a thick line segment: `stroke: var(--color-text-secondary)`, stroke-width: 2, y position just below X-axis.
- Label with voice type name.

Current f0 cursor (D-12):
- Vertical line at `xScale(f0)`: `stroke: var(--color-accent)`, stroke-width: 2, full plot height.

Current F1 marker:
- Horizontal dashed line at `yScale(f1Freq)`: `stroke: var(--color-f1)`, stroke-width: 1.5, stroke-dasharray: 4,3.

Active strategy highlight:
- If `r1Strategy` is set and `strategyMode !== 'off'`, highlight the corresponding diagonal line with thicker stroke (2px) and full opacity.

Chart title (UI-SPEC): `<text>` at top: "R1 (First Resonance)", font-size 12px, font-weight 600, fill `var(--color-text)`.

Use `preserveAspectRatio="none"` on the SVG so it stretches to fill its grid cell. Set `width="100%" height="100%"` on the SVG element, with viewBox computed from the width/height props.

All colors reference CSS custom properties defined in app.css.
